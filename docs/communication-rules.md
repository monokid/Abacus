# Communication Rules

The user wants to stay in the loop as Abacus progresses.

## Standing Rule

At each milestone, report:

- Where we are now.
- What changed.
- What was verified.
- What is incomplete.
- What the next step is.
- Which key decisions need user input.

## Key Decisions Need Conversation

Always pause or explicitly call out decisions that affect:

- Stack or architecture.
- Data model shape.
- User-facing workflows.
- Visual direction.
- Persistence, backup, recovery, import, export, or update behavior.
- Public release posture.
- Anything that would be hard to reverse later.

## Preferred Style

Keep updates plain and practical. The user should never have to infer whether we are
planning, building, verifying, or asking for a decision.
